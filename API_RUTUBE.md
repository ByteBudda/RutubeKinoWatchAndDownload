# 🛸 Rutube API: The Complete Reverse Engineering Guide
Эта документация описывает внутреннее устройство API Rutube по состоянию на апрель 2026 года. Она предназначена для разработки независимых плееров, альтернативных фронтендов и систем архивации контента.
## 1. Архитектура и Глобальные параметры
API Rutube построено на принципах REST, но имеет сильную привязку к динамическим конфигурациям интерфейса (Backend-Driven UI).
 * **Base URL:** https://rutube.ru/api/
 * **Default Headers:**
   * User-Agent: Рекомендуется использовать заголовок от Smart TV (Tizen/WebOS) для получения наиболее «чистых» потоков видео.
   * Accept: application/json
 * **Пагинация:** Почти все списки поддерживают:
   * ?page=N — номер страницы (с 1).
   * ?limit=N или ?per_page=N — количество элементов (обычно до 100).
   * Ответ содержит поле next (прямая ссылка на следующую страницу) и has_next (boolean).
## 2. Видео-контент (The Core)
### 2.1 Получение данных видео
GET /api/video/{video_id}/
Основной эндпоинт для карточки видео.
 * **Важные поля:** * is_official / is_licensed — флаги для фильтрации пиратского контента.
   * duration — длительность в секундах.
   * author — объект с id, name и avatar_url.
### 2.2 Стриминг и Плеер (Get Play Options)
GET /api/video/{video_id}/get_play_options/
**Критический эндпоинт для альтернативного клиента.**
 * Возвращает доступные форматы (HLS/m3u8, DASH).
 * Позволяет получить прямые ссылки на потоки в обход стандартного плеера.
 * **Параметр quality:** позволяет запрашивать конкретное разрешение.
## 3. Навигация и "Матрешки" (Feeds & Groups)
Rutube использует сложную систему вложенных контейнеров. Чтобы добраться до видео, нужно "развернуть" цепочку.
### 3.1 Фиды (Feeds)
GET /api/feeds/{slug}/ (например, movies-serials, tnt, kids)
Возвращает структуру страницы:
 * tabs — массив вкладок.
 * resources — ссылки внутри вкладок, ведущие на cardgroup или person (каналы).
### 3.2 Группы карточек (Card Groups)
GET /api/feeds/cardgroup/{id}/
Это основной контейнер для лент на главной.
 * Если в ответе есть results, это прямой плейлист.
 * Если есть ссылки на другие API — это "папка", которую нужно открыть.
## 4. Телешоу и Сериалы (Metainfo)
В отличие от обычных видео, сериалы имеют строгую иерархию:
**Show -> Season -> Episode.**
 1. **Каталог шоу:** GET /api/metainfo/tv/
 2. **Детали шоу:** GET /api/metainfo/tv/{id}/
 3. **Сезоны:** GET /api/metainfo/tv/{id}/seasons/
 4. **Список видео (серий):** GET /api/metainfo/tv/{id}/video/?season={N}
## 5. Поиск и Социальные функции
### 5.1 Поиск
GET /api/search/video/?query={text}
 * sort=rank — релевантность.
 * sort=hits — популярность.
 * sort=created — новинки.
 * filter=duration:short — видео до 5 минут.
### 5.2 Каналы (Person)
GET /api/video/person/{user_id}/
Возвращает ленту видео конкретного автора.
 * **Лайфхак:** Для получения только "Reels" (коротких видео) часто используются специфические теги или фильтры в запросе к персоне.
## 6. Player PostMessage API (Управление Iframe)
Если твой клиент использует <iframe> оригинального плеера, ты можешь управлять им через JS:
```javascript
// Пример команды: Перемотка
iframe.contentWindow.postMessage(JSON.stringify({
  type: 'player:setCurrentTime',
  data: { currentTime: 120 }
}), '*');

```
**Доступные события (Listen):**
 * player:currentTime — текущая позиция (тик раз в секунду).
 * player:changeState — (playing, paused, ended).
 * player:error — ошибки воспроизведения или геоблокировка.
## 7. Технические ограничения (Safety First)
 1. **Rate Limits:** Rutube очень чувствителен к парсингу. При превышении 10-15 запросов в секунду прилетает 429 Too Many Requests.
   * *Решение:* Использовать p-limit в Node.js или очередь с задержкой в 600-1000мс.
 2. **CORS:** API Rutube не всегда отдает правильные заголовки для браузерных fetch.
   * *Решение:* В альтернативном клиенте (Electron/Garuda Linux) делать запросы через бэкенд (Node.js) или отключать web-security.
 3. **Авторизация:** Для лайков и подписок требуется POST на /transport/ с получением сессионной куки.
### Статусы ответов (Reference)
 * 200 OK — Данные получены.
 * 401/403 — Контент заблокирован в вашем регионе или требует подписки "Top".
 * 404 — Видео удалено или ссылка устарела.
 * 429 — Слишком много запросов (нужна пауза).
