1. Обзор

Rutube предоставляет несколько уровней API:

  • Content API — получение контента (витрины, видео, плейлисты, каналы, поиск, категории);
  • Player API — управление встроенным плеером;
  • oEmbed API — встраивание видео по стандарту oEmbed;
  • Partner API — загрузка и управление видео (требует авторизации).

API возвращает данные в форматах JSON, JSONP и XML. Основной базовый URL — https://rutube.ru/api.

2. Content API

2.1. Витрины (Feeds)

Витрины — способ организации и представления видеоконтента. Каждая витрина состоит из вкладок, содержащих источники карточек (теги, ТВ-шоу, пользователи и т.д.).

Эндпоинт: GET https://rutube.ru/api/feeds/{slug}?format=json

ПараметрТипОписание
slugstringУникальный идентификатор витрины (например, movies, series, tnt)
formatstringФормат ответа (json, jsonp, xml, api для human-readable)

Пример ответа (основные поля):

{
  "id": 1,
  "slug": "movies",
  "name": "Фильмы",
  "meta_description": "...",
  "page_url": "https://rutube.ru/feeds/movies/",
  "tabs": [ ... ]
}

Поля ответа витрины:

ПолеТипОписание
idnumberID витрины
slugstringИдентификатор витрины
namestringНазвание
meta_descriptionstringОписание
page_urlstringАдрес на rutube.ru
tabsarrayМассив вкладок

Поля вкладки (tabs[]):

ПолеТипОписание
idnumberID вкладки
namestringНазвание
sortstringРекомендация сортировки (created_date, original, random)
order_numbernumberПорядковый номер
slugstring|nullУникальное название для hash-навигации
linkstring|nullВнешняя ссылка (если задана, вкладка делает редирект)
resourcesarrayИсточники карточек

Поля ресурса (resources[]):

ПолеТипОписание
idnumberID источника
object_idstringСлужебный ID
content_typeobjectТип источника (model: tag, tv, cardgroup, userchannel и др.)
urlstringАдрес для загрузки карточек
namestringНазвание источника
extra_paramsobjectДополнительные параметры

2.2. Типы источников (content_type.model)

МодельТип контентаОписание
tagСписок видеоТег/подборка видео
tvТВ-шоуСериал или телешоу
tvlastepisodeПоследний эпизодПоследний эпизод ТВ-шоу
playlistПлейлистСписок видео (плоская структура)
cardgroupКардгруппаСмешанный контейнер (видео, сериалы, каналы)
subscriptiontvseriesПодпискаПартнёрская подборка (структура как cardgroup)
subscriptionfilmsПодписка на фильмыПодборка фильмов по подписке (аналогично cardgroup/subscriptiontvseries)
promogroup / promofeedПромоПромо-элементы (не видео)
userchannelКаналПользователь Rutube
personПерсонаАктёр, режиссёр и т.д.
feedsourceВнешняя ссылкаБаннер или редирект

Примечание по subscriptionfilms: в витрине moviesgenres ресурс с model=subscriptionfilms ведёт на /api/feeds/cardgroup/1428, т.е. это обёртка над CardGroup с определёнными параметрами.

2.3. Список видео (Теги / Плейлисты)

Эндпоинт: GET https://rutube.ru/api/tags/video/{tag_id}/?page=1&limit=20&format=json

ПараметрТипОписание
pagenumberНомер страницы
limitnumberКоличество элементов
sortstringСортировка (tagged_a, tagged_d, created_a, created_d, publication_a, publication_d)

Пример ответа (плоская структура):

{
  "results": [
    {
      "id": "abc123",
      "type": "video",
      "title": "Название",
      "duration": 7234,
      "thumbnail_url": "https://...",
      "author": { "name": "Канал", "id": 456 },
      "hits": 1500000,
      "publication_ts": 1672531200,
      "is_paid": false,
      "pg_rating": { "age": 16 }
    }
  ],
  "has_next": true,
  "next": "/api/tags/video/7487/?page=2",
  "page": 1,
  "per_page": 20
}

Поля элемента списка (видео): id, type, title, duration, thumbnail_url, preview_url, author, hits, publication_ts, is_paid, pg_rating, description.

2.4. Кардгруппы (Смешанные списки)

Эндпоинт: GET https://rutube.ru/api/feeds/cardgroup/{id}/?format=json

Каждый элемент содержит content_type.model и вложенный object. Может включать tv, userchannel, tag.

{
  "results": [
    {
      "content_type": { "model": "tv" },
      "object": {
        "id": 1478618,
        "name": "Сериал XYZ",
        "poster_url": "...",
        "year_start": 2021,
        "kinopoisk_rating": 7.8
      }
    }
  ],
  "has_next": false
}

2.5. Метаинформация ТВ-шоу

Эндпоинт: GET https://rutube.ru/api/metainfo/tv/{tv_id}/?format=json

Эпизоды: GET https://rutube.ru/api/metainfo/tv/{tv_id}/video/?page=1&sort=series_d&format=json
Параметры сортировки: series_a, series_d, created_a, created_d, publication_a, publication_d. Фильтры: season, episode, origin__type (rtb, rst, ytb, vim, now).

2.6. Видео пользователя (Канал)

GET https://rutube.ru/api/video/person/{person_id}/?page=1&limit=20&format=json

2.7. Метаданные видео (расширенная структура)

GET https://rutube.ru/api/video/{video_id}/

Возвращает детальную информацию: id, title, description, thumbnail_url, duration, hits, author, category, all_tags, restrictions, feed_url, feed_name, episode, season, tv_show_id, hide_comments, hide_likes, hide_dislikes, embed_url, html, persons, genres, show и многие другие (подробнее в полном ответе).

2.8. Поиск видео

GET https://rutube.ru/api/search/video/?query={query}&page=1&limit=20&format=json
Поддерживает фильтры: created, author_id, category_id, duration.

2.9. Параметры воспроизведения (Play Options)

GET https://rutube.ru/api/play/options/{video_id}/

Возвращает конфигурацию плеера: acl_access, black_rabbit, appearance (цвет, элементы управления), advert (массив рекламных блоков с url_template, start, count и т.д.), captions (субтитры), author, category, description. Полный перечень полей описан в разделе документации.

2.10. Пагинация

Все списки содержат поля: results, page, has_next, next, per_page.

2.11. Категории видео (полный список)

GET https://rutube.ru/api/video/categories/

Возвращает массив категорий с полями: id, category_url, related_showcase, update_ts, name, short_name, for_kids, for_import. Полная таблица категорий (ID, название, slug, связанная витрина) приведена в документации.

2.12. Связанные эндпоинты метаинформации

  • Персоны видео: GET /api/metainfo/video/{video_id}/videoperson
  • Жанры видео: GET /api/metainfo/video/{video_id}/videogenre
  • Информация о ТВ-шоу: GET /api/metainfo/contenttvs/{video_id}
  • Управление подпиской на ТВ-шоу: POST (или DELETE) /api/subscription/card/tv/{tv_show_id} — GET запрещён (возвращает 405). Требует аутентификации.

2.13. Витрина жанров кино (moviesgenres)

GET https://rutube.ru/api/feeds/moviesgenres

Это обычная витрина со структурой Feed, но её вкладки представляют кинематографические жанры: Комедия, Драма, Боевик, Приключения, Мелодрама, Семейный, Исторический, Триллер, Фантастика, Военный, Фэнтези, Детектив и др. Каждая вкладка содержит ресурсы типа tag и feedsource (баннеры). Slug’и вкладок соответствуют английским названиям жанров (comedy, drama, action, adventure и т.д.). Вкладки могут включать ресурсы subscriptionfilms, ведущие на cardgroup.

3. oEmbed API

GET https://rutube.ru/api/oembed/?url={video_url}&format=json
Возвращает стандартный oEmbed-ответ с HTML для встраивания.

4. Player API

Встраивание через iframe: https://rutube.ru/play/embed/{video_id}/. Управление через postMessage: play, pause, stop, seek, setVolume, changeVideo, setSkinColor и другие. События: player:ready, player:playStart, player:changeState, player:currentTime и др. (подробно в полной версии).

5. Partner API (авторизованный доступ)

Требуется базовая HTTP-авторизация. Методы: загрузка видео, обновление, удаление, управление обложками, плейлистами. Подробнее в официальной документации v1.

6. Тонкости и особенности

  • Плоская vs вложенная структура: теги/плейлисты — плоская, cardgroup/subscriptiontvseries/subscriptionfilms — вложенная (item.object).
  • Превью: размеры ?size=l, m, s (пример: https://pic.rutube.ru/video/…?size=m).
  • CORS: необходим прокси-сервер.
  • Коды ошибок: 400, 401, 403, 404, 405, 500.
  • Сортировка: для tv/person — series_a и т.д., для tag — tagged_a и т.д., для cardgroup/userchannel — ordering (publication_ts, created_ts, views, id).

7. Примеры запросов

  • Витрина: GET /api/feeds/movies?format=json
  • Тег: GET /api/tags/video/7487/?page=1&format=json
  • Видео: GET /api/video/b07639337aa57ce56578bbb9e40e0102/
  • Поиск: GET /api/search/video/?query=комедия
  • Канал: GET /api/video/person/123/
  • Эпизоды: GET /api/metainfo/tv/28/video?season=1
  • Play Options: GET /api/play/options/{id}/
  • Категории: GET /api/video/categories/
  • Жанры кино: GET /api/feeds/moviesgenres
